## 9. 建议

### P1 — 重要

- **SOCKS5 代理稳定性**：`brd.superproxy.io` 在 02:15–07:19 间歇性拒绝连接（75 次错误）。虽然故障转移兜住了，但每次重试增加约 3s 延迟。建议：
  - 检查 Bright Data 代理配额/规则配置
  - 考虑为 `abc` 账户配置备用代理
  - 或评估是否所有账户都需要 SOCKS5 代理

### P2 — 改进

- **mt1 账户**：07:39 因无 OAuth 令牌被自动禁用。若此账户有效，需执行 `ccproxy oauth login mt1` 完成授权。若已弃用，建议从账户注册表中移除。

### P3 — 可观察

- **冷窗口节流**：8 次误触（`requests ≤ 5`），对用户无影响（队列等待后成功）。`minThrottleSamples=3` 的 guard 在边界处仍有少量漏网，可考虑提高到 5。
- **7d 利用率 62%**：距离限制有余量，但如果流量持续增长需关注。当前日均约 4,200 请求，以 3 账户分摊可持续。
