1登录仪表盘

管理员会提供你的用户名和密码。打开仪表盘地址，在登录界面输入凭据即可进入。

2添加 Claude 账户

登录后，你需要添加至少一个 Claude 账户。每个账户对应一个 Anthropic OAuth 授权。

1
点击右上角的 + Add Claude 按钮
2
在弹出的输入框中输入账户名称（如 my-claude-1），名称自定义，仅用于在列表中区分不同账户
3
点击 Save 保存，或按 Enter 确认
提示
你可以添加多个账户。

3OAuth 登录认证

添加账户后，需要通过 OAuth 授权让 ccproxy 获取 Claude API 的访问令牌。这是最关键的一步。

第 1 步：点击 Login，打开授权链接

1
在账户行中点击 Login 按钮，弹出 OAuth 登录弹窗
2
点击 Open 在新标签页中打开授权页面（或点击 Copy 复制链接手动打开）

第 2 步：在 Anthropic 页面中授权

3
在打开的 Anthropic 页面中 登录你的 Claude 账户
4
登录后会显示权限确认页面，查看权限列表后点击 Authorize 按钮完成授权

第 3 步：复制授权码，粘贴回仪表盘

5
授权成功后会显示 Authentication Code 页面，点击 Copy Code 复制授权码
6
回到仪表盘弹窗，将授权码 粘贴到第二行输入框（不是第一行的 URL），点击 Submit
注意
OAuth 会话有效期仅 10 分钟。如果超时未完成，请关闭弹窗后重新点击 Login 开始新的认证流程。
提示
认证成功后，Token 状态会变为 valid。系统会在令牌过期前自动刷新，通常无需手动操作。

4设置代理（可选）

如果你的 Claude 账户需要通过 SOCKS5 代理访问 Anthropic API，可以为每个账户单独配置代理。

1
桌面端：双击账户行的 Proxy 列进入编辑模式；移动端：点击 Edit 按钮
2
输入 SOCKS5 代理地址，格式：socks5://用户名:密码@IP:端口
3
按 Enter 保存，或按 Esc 取消
提示
如果可以直接访问 Anthropic API，不需要设置代理，留空即可。清空代理地址并按 Enter 可以移除已配置的代理。

5测试账户连通性

设置完成后，点击 Test 按钮验证账户是否能正常连接 Anthropic API。

1
在账户行中点击 Test 按钮
2
显示 ✓ OK 表示连接正常；显示 ✗ Fail 时点击按钮可查看错误原因
注意
如果测试失败，请检查：OAuth 是否已完成登录、代理地址是否正确、账户是否被禁用。

6Telegram 通知（可选）

配置 Telegram 通知后，当你的账户被禁用或封禁时，会自动收到消息提醒。

1
在 Telegram 中找到 @BotFather，发送 /newbot 创建机器人，获取 Bot Token
2
获取你的 Chat ID：将机器人拉入群组后，访问 https://api.telegram.org/bot你的Token/getUpdates 查看 chat id；或者私聊机器人后查看
3
将 Bot Token 和 Chat ID 填入仪表盘，点击 保存配置
4
点击 发送测试消息 验证是否能正常接收

7理解状态指示器

仪表盘中的各种状态标签和进度条帮助你了解账户的运行状况。

账户健康状态

Healthy 正常运行
Cooling down 触发限速，冷却中
Banned 被平台封禁
Disabled 已禁用（连续 401 等）

Token 状态

valid 令牌有效，正常工作
expiring soon 即将过期，系统会自动刷新
expired 已过期，需要重新 Login
no token 未认证，需要 OAuth Login

预算用量

每个账户有两个滚动窗口的用量追踪：5 小时窗口和 7 天窗口。

低用量
正常使用，无需关注
中等用量
接近限额，系统开始限制调度
高用量
接近或达到限额

预算状态

Normal 正常调度，无限制
Sticky 仅允许已绑定的会话使用
Blocked 达到限额，暂停调度
提示
预算用量会随时间自动恢复。5 小时窗口恢复较快，7 天窗口恢复较慢。悬停在进度条上可以看到预计重置时间。

常用操作按钮

Login 发起 OAuth 授权
Test 测试账户连通性
Refresh 强制刷新令牌
Remove 删除账户及令牌
Enable 重新启用被禁用的账户
